# 🧭 Skill Map: Core OpenClaw Capabilities

This map indexes key skills, categorized by function. Use these skill names for quick recall.

## 📁 1. Productivity & Writing
*   **obsidian:** Manages notes and knowledge graph integration.
*   **apple-notes / bear-notes:** Native integration for Apple's note-taking ecosystem.
*   **daily-briefing:** Automates the generation and delivery of morning and evening status reports.
*   **total-recall-search:** High-powered search across memory, enabling deep knowledge retrieval.
*   **memory_get / memory_search:** Direct interface to long-term, cross-session memory recall.

## 💻 2. System & Development Operations
*   **github:** Manages GitHub interactions (issues, PRs, commits).
*   **gh-issues:** Dedicated tool for specific GitHub issue workflows.
*   **aws-deploy:** Orchestrates deployments to AWS services.
*   **exec / process:** Core system tools for shell command execution, backgrounding, and process management.
*   **cron:** Handles all scheduled, recurring tasks (alerts, cleanup, syncs).

## 📈 3. Data & Analytics
*   **ga4-analytics:** Collects and analyzes Google Analytics data.
*   **expense-tracker-pro / personal-finance:** Specialized tools for managing financial data.
*   **xlsx / docx / pptx:** Handling various document formats for creation and editing.
*   **web-perf:** Tools for analyzing website performance metrics.

## 🌐 4. Communication & Connectivity
*   **gmail-send:** Primary method for sending emails using authenticated OAuth accounts.
*   **slack:** Interacts with Slack for team communication.
*   **imsg:** Dedicated messaging interface (e.g., for internal system alerts).

***
## ⚙️ Maintenance/Updates
This section tracks the processes and scripts used to keep the Skill Map accurate and up-to-date.
*   **Skill List Source:** The primary source of truth for all skills is the system's installed directories.
*   **Manual Update Procedure:** Follow the steps in `scripts/update_skill_map.sh`.
*   **Automated Discovery:** For major additions, run `openclaw skills check` output and consult the dedicated directory structure.
