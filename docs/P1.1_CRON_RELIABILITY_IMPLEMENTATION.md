# P1.1: Cron Job Reliability Implementation

## Overview
Add explicit timeouts and failure alerting to all 17 cron jobs to prevent scheduler starvation.

## Current Status
- 15 active cron jobs configured
- No explicit timeouts (inherits default)
- No failure alerting mechanism
- Risk: If any job hangs >30min, scheduler stalls

## Implementation Plan

### Step 1: Timeout Configuration (20-30 min per job)
Each job needs explicit `timeoutSeconds` parameter:
- Morning Briefing: 5 min (usually <2 min)
- Evening Briefing: 5 min
- API Quota Checks: 3 min (API calls only)
- GPU Health Check: 10 min (SSH + inference test)
- Session Reset: 5 min (cleanup only)
- Weekly/Monthly jobs: 15-20 min (larger workloads)

### Step 2: Failure Alerting
When a job fails or times out:
- Send Telegram alert to Bob
- Log to cron_runs/ with status + error
- Include: job name, scheduled time, actual time, duration, error

### Step 3: Queue Monitoring
Add health check script:
- Monitor cron queue depth
- Alert if >2 jobs pending (backlog)
- Check for stuck jobs (running > timeout)
