## Pre-Update Checklist

### 24 Hours Before Update

- [ ] **Review Release Notes**
  - Go to: https://github.com/openclaw/openclaw/releases
  - Check for known issues in target version
  - Read migration guide (if any)

- [ ] **Check Current System Status**
  ```bash
  openclaw --version
  openclaw gateway status
  openclaw cron list
  ```

- [ ] **Document Current State**
  - Use: `bash ~/.openclaw/workspace/docs/P2.1_CURRENT_SYSTEM_STATUS.md`
  - Record your OpenClaw version
  - Note any recent customizations

### 1 Hour Before Update

- [ ] **Backup Production System**
  ```bash
  bash ~/.openclaw/workspace/scripts/backup-before-update.sh
  ```
  - Confirms backup location
  - Verifies backup integrity
  - Creates manifest for restore

- [ ] **Setup Staging Environment**
  ```bash
  bash ~/.openclaw/workspace/scripts/setup-staging-environment.sh
  ```
  - Creates isolated test environment
  - Copies minimal configuration
  - Ready for test update

- [ ] **Test Update in Staging** (Optional but recommended)
  ```bash
  export OPENCLAW_HOME=~/.openclaw-staging
  openclaw update
  bash ~/.openclaw-staging/test-staging.sh
  ```
  - If staging update fails → don't update production
  - If staging passes → safe to proceed

- [ ] **Stop Automation Jobs** (if updating during work hours)
  ```bash
  # Disable jobs to prevent conflicts
  openclaw cron disable "Morning Briefing"
  openclaw cron disable "Evening Briefing"
  ```

### Immediate Pre-Update

- [ ] **Close All OpenClaw Sessions**
  - Stop any active agents
  - Save important work
  - Close terminal windows

- [ ] **Stop Gateway**
  ```bash
  openclaw gateway stop
  ```
  - Confirm it's stopped
  - Check: `openclaw gateway status` should show "stopped"

- [ ] **Final Backup Verification**
  ```bash
  ls -lh ~/.openclaw/backups/pre-update-*/
  ```
  - Confirm latest backup exists
  - Note backup directory name

## Update Procedure

### Step 1: Perform Update

```bash
# Option A: Update via Homebrew (recommended)
brew upgrade openclaw

# Option B: Update via openclaw CLI
openclaw update

# Option C: Update to specific version
brew install openclaw@2026.3.25  # Example version
```

### Step 2: Monitor Update Progress

- Watch console output
- Look for success messages
- DO NOT INTERRUPT (let it finish completely)

### Step 3: Check Update Status

```bash
openclaw --version
# Should show new version number
```

## Post-Update Testing

### Immediate Tests (Critical)

```bash
# 1. Start gateway
openclaw gateway start

# 2. Check gateway status
openclaw gateway status
# Look for: "RPC probe: ok"

# 3. List cron jobs
openclaw cron list
# Should show your jobs with timeouts

# 4. Test a simple command
read ~/.openclaw/openclaw.json
# Should work without errors
```

### Run Comprehensive Test Suite

```bash
bash ~/.openclaw/workspace/scripts/test-post-update.sh
```

This tests:
- ✅ Gateway functionality
- ✅ Configuration loading
- ✅ Cron job parsing
- ✅ Tool availability
- ✅ Integration status
- ✅ Security settings

## Rollback Procedure

### If Update Fails During Installation

```bash
# 1. Stop gateway
openclaw gateway stop

# 2. Identify last good backup
ls -lt ~/.openclaw/backups/ | head -1
# Copy the timestamp (YYYYMMDD_HHMMSS)

# 3. Backup broken state (for debugging)
mv ~/.openclaw ~/.openclaw-broken-$(date +%s)

# 4. Restore from good backup
cp -r ~/.openclaw/backups/pre-update-YYYYMMDD_HHMMSS/config/* ~/.openclaw/

# 5. Restore cron jobs
cp ~/.openclaw/backups/pre-update-YYYYMMDD_HHMMSS/cron/jobs.json ~/.openclaw/cron/

# 6. Restart gateway
openclaw gateway start

# 7. Verify
openclaw gateway status
```

### If Update Succeeds But Tests Fail

```bash
# 1. Stop gateway
openclaw gateway stop

# 2. Downgrade to previous version
# Find previous version in release notes
brew install openclaw@2026.3.8  # Example

# 3. Restore configuration
cp -r ~/.openclaw/backups/pre-update-YYYYMMDD_HHMMSS/config/* ~/.openclaw/
cp ~/.openclaw/backups/pre-update-YYYYMMDD_HHMMSS/cron/jobs.json ~/.openclaw/cron/

# 4. Restart gateway
openclaw gateway start

# 5. Verify
openclaw gateway status
bash ~/.openclaw/workspace/scripts/test-post-update.sh
```

### Cleanup After Rollback

```bash
# Remove broken installation (keep for debugging if needed)
rm -rf ~/.openclaw-broken-*

# Remove staging environment
rm -rf ~/.openclaw-staging

# Re-enable automation jobs
openclaw cron enable "Morning Briefing"
openclaw cron enable "Evening Briefing"
```

## Emergency Recovery

### If Gateway Won't Start

```bash
# Check logs
tail -50 ~/.openclaw/logs/gateway.log

# Try restarting daemon
launchctl stop ai.openclaw.gateway
sleep 2
launchctl start ai.openclaw.gateway

# If still fails, restore from backup
openclaw gateway stop
cp -r ~/.openclaw/backups/pre-update-LATEST/config/* ~/.openclaw/
openclaw gateway start
```

### If Cron Jobs Are Lost

```bash
# Restore jobs from backup
cp ~/.openclaw/backups/pre-update-LATEST/cron/jobs.json ~/.openclaw/cron/

# Verify
openclaw cron list
```

### If Configuration Is Corrupted

```bash
# Restore both configs
cp ~/.openclaw/backups/pre-update-LATEST/config/openclaw.json ~/.openclaw/
cp ~/.openclaw/backups/pre-update-LATEST/config/config.json ~/.openclaw/

# Restart gateway
openclaw gateway restart
```

### Complete System Restore

If everything is broken and you need a full restore:

```bash
# 1. Identify last known good backup
BACKUP_DIR=~/.openclaw/backups/pre-update-YYYYMMDD_HHMMSS

# 2. Stop everything
openclaw gateway stop

# 3. Rename broken system
mv ~/.openclaw ~/.openclaw-broken-$(date +%s)

# 4. Restore entire state from backup
cp -r "$BACKUP_DIR/config"/* ~/.openclaw/
mkdir -p ~/.openclaw/cron
cp "$BACKUP_DIR/cron/jobs.json" ~/.openclaw/cron/
mkdir -p ~/.openclaw/workspace
cp -r "$BACKUP_DIR/workspace"/* ~/.openclaw/workspace/ || true

# 5. Start fresh
openclaw gateway start

# 6. Verify
bash ~/.openclaw/workspace/scripts/test-post-update.sh
```

## Troubleshooting

### "Gateway won't start after update"

**Cause:** Configuration incompatibility  
**Solution:**
1. Check logs: `tail ~/.openclaw/logs/gateway.log`
2. Restore config: `cp ~/.openclaw/backups/pre-update-LATEST/config/* ~/.openclaw/`
3. Try again: `openclaw gateway start`

### "Cron jobs disappeared"

**Cause:** Job storage format changed  
**Solution:**
1. Restore from backup: `cp ~/.openclaw/backups/pre-update-LATEST/cron/jobs.json ~/.openclaw/cron/`
2. Verify: `openclaw cron list`
3. If different format, manually recreate jobs from notes

### "Tools not working after update"

**Cause:** Tool permissions reset  
**Solution:**
1. Check which tools fail: `bash ~/.openclaw/workspace/scripts/test-post-update.sh`
2. Verify tool configuration: `grep -A5 '"tools"' ~/.openclaw/config.json`
3. Restore config if needed: `cp ~/.openclaw/backups/pre-update-LATEST/config/config.json ~/.openclaw/`

### "Update takes too long"

**Normal:** Updates can take 5-15 minutes  
**Timeout:** If >30 min, check:
1. Disk space: `df -h /`
2. Network: `ping github.com`
3. System load: `top` (Ctrl+C to exit)

If truly stuck:
1. Kill hanging process (research carefully first!)
2. Restore from backup
3. Try update again
