## Problem Statement

After OpenClaw updates, tool permissions sometimes fail to restore:
- exec tool becomes unavailable
- read/write tools restricted
- File operations blocked
- Manual re-enable required

**Impact:** Breaks automation workflows until manually fixed

**Solution:** Document current allowlist + create restoration script

## Post-Update Verification Checklist

After any OpenClaw update, verify tools are working:

```bash
#!/bin/bash
# Quick tool verification

echo "Testing tool availability after update..."
echo ""

# Test exec
echo "1. exec tool..."
if bash -c "echo 'test' > /tmp/tool-test && rm /tmp/tool-test" 2>/dev/null; then
  echo "  ✅ exec working"
else
  echo "  ❌ exec broken"
fi

# Test read
echo "2. read tool..."
if [ -r ~/.openclaw/openclaw.json ]; then
  echo "  ✅ read working"
else
  echo "  ❌ read broken"
fi

# Test write
echo "3. write tool..."
if bash -c "echo 'x' > /tmp/w-test && rm /tmp/w-test" 2>/dev/null; then
  echo "  ✅ write working"
else
  echo "  ❌ write broken"
fi

# Test edit
echo "4. edit tool..."
if [ -w ~/.openclaw/workspace/TOOLS.md ]; then
  echo "  ✅ edit working"
else
  echo "  ❌ edit broken"
fi

# Test cron
echo "5. cron tool..."
if openclaw cron list > /dev/null 2>&1; then
  echo "  ✅ cron working"
else
  echo "  ❌ cron broken"
fi

# Test web tools
echo "6. web_search..."
if grep -q "brave" ~/.openclaw/openclaw.json; then
  echo "  ✅ brave configured"
else
  echo "  ❌ brave not configured"
fi

# Test denied tools (should NOT work)
echo ""
echo "Security verification (should all be denied):"
echo "7. camera.snap - should be denied"
echo "8. screen.record - should be denied"
echo "9. sms.send - should be denied"
```

## Permission Restoration Procedure

### If Tools Break After Update

**Step 1: Identify which tool broke**
```bash
bash ~/.openclaw/workspace/scripts/verify-tools.sh
```

**Step 2: Restore configuration**
```bash
# Restore tools allowlist from backup
cp ~/.openclaw/backups/pre-update-LATEST/config/config.json ~/.openclaw/

# Or manually restore if backup missing:
# See "Manual Restoration" section below
```

**Step 3: Restart gateway**
```bash
openclaw gateway restart
```

**Step 4: Verify tools working**
```bash
bash ~/.openclaw/workspace/scripts/verify-tools.sh
```

## Sudo Access Whitelisting

Your system has passwordless sudo access for specific commands.
This is security-by-whitelist, not blanket sudo.

**Location:** `/etc/sudoers.d/momotaro`

**Whitelisted Commands:**
- softwareupdate (system updates)
- brew (package management)
- launchctl (service management)
- systemctl (service control)
- dscacheutil (DNS cache)
- clawhub (skill management)
- Debugging tools

**Usage:**
```bash
# These work without password (security-vetted):
sudo brew install package
sudo launchctl load ~/Library/LaunchAgents/service.plist
sudo softwareupdate -i -a

# For any other sudo command, password is required
```
