## Timeout Rationale

**Why these specific timeouts?**

1. **Short jobs (3-5 min)** — API calls, reporting, reminders
   - Typically complete in <2 minutes
   - Timeout provides 2-3x safety buffer

2. **Medium jobs (10 min)** — System updates, deployments, SSH operations
   - May include network calls or builds
   - Timeout accounts for network variability

3. **Longer jobs (15-20 min)** — Analytics, complex processing, strategy planning
   - Involve more computation
   - Timeout prevents indefinite hangs

4. **Queue monitoring** — Alert if >2 jobs running simultaneously
   - Normal: 0-1 concurrent jobs
   - Warning: 2+ concurrent jobs (backlog forming)
