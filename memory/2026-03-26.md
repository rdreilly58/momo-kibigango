## Key Learnings & Decisions

### Cost Optimization Strategy
- **Direct is best when cheap:** Haiku doesn't need routing (already optimal)
- **OpenRouter Auto for medium/complex:** Saves 30-40% on Opus/GPT-4
- **Right tool for right job:** Better quality + better cost
- **Track everything:** Cost data enables future optimization

### Implementation Approach
- **Phase architecture:** Tier A (classify) → Tier B (route) → Tier C (batch)
- **Progressive delivery:** Each tier independent, builds on previous
- **Testing first:** Every component tested before production
- **Documentation as code:** SOUL.md rules = source of truth

### Fallback Strategy
- Haiku: Direct (no fallback needed, already cheapest)
- Opus: OpenRouter Auto → Opus → Haiku
- GPT-4: OpenRouter Auto → Opus → Haiku
- **3-tier chain ensures reliability**
