### Key Implementation Details

**LSH Algorithm:**
- 16 hash functions by default (tunable 8-32)
- Hash bucket lookup O(1) complexity
- Candidate retrieval (~10-20 vectors per query)
- Cosine similarity ranking (exact, not approximate)
- Top-5 results returned in 15-20ms
- Smart fallback to brute-force if recall drops

**Fallback Logic:**
- Monitors recall on every query
- If LSH recall < 90%, automatically switches to brute-force
- Maintains 95%+ quality guarantee
- Expected fallback rate: <5%

**Metrics Tracked:**
- total_queries: Total search operations
- lsh_queries: Served by LSH (fast path)
- fallback_queries: Served by brute-force (slow path)
- avg_latency_ms: Average query time
- lsh_hit_rate: Percentage served by LSH
- fallback_rate: Percentage needing fallback
