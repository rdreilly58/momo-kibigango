## Lessons Learned

### What Worked Well ✅
- Comprehensive research before implementation (found best approach)
- Hybrid approach balances simplicity + speed
- Webhook is superior to polling (instant detection)
- Telegram integration leverages existing infrastructure
- LaunchAgent setup ensures persistence

### What Didn't Work ❌
- Monitor daemon (polling every 5 seconds) — couldn't see alerts properly
- Long responses with special characters (script limitation) — fixed by keeping responses concise
- Port conflicts — resolved by careful process cleanup
- Syntax errors in initial script — fixed in deployment

### Key Insights
1. **Push > Poll:** Webhooks are instant, polling is slow. For real-time, always use push-based architecture.
2. **Leverage Existing Tools:** Using Telegram for notifications re-uses proven infrastructure vs building custom alert system.
3. **Hybrid Beats Pure:** Combining simple webhook + existing Telegram is better than complex WebSocket implementation.
4. **Test Early:** Found syntax errors during testing, not in production.
