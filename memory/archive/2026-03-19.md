**Architecture:**
```
Draft (Qwen2-1.5B) → Qualifier (Phi-2 2.7B) → Target (Qwen2-7B-4bit)
Stage 1 acceptance: 10% threshold
Stage 2 acceptance: 3% threshold
Result: 76.5% tokens bypass target model
```

**Why 3-Model Better:**
- Hierarchical verification (more elegant)
- Higher acceptance rate (76.5% vs 74.6%)
- Better VRAM efficiency (11.6GB for 3-model vs 10.8GB for 2-model = better tokens/GB)
- Proves PyramidSD paper concepts in production
