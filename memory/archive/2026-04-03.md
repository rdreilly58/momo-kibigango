## CRITICAL LESSON: Speculative Decoding ≠ Model Routing

**The fundamental mistake in v1:**
- I built a model selection/routing system and called it "speculative decoding"
- Speculative decoding is a specific algorithm that operates at the TOKEN level
- It exploits GPU parallelism: verifying K tokens costs same as generating 1
- The speedup comes from parallel verification, NOT from choosing cheaper models
- API models (Claude, GPT) cannot do this — no logit access, no parallel verification
- This requires LOCAL models with shared tokenizer and direct logit access

**Bob caught this immediately.** Future sessions: when implementing a paper, implement what the paper actually describes, don't adapt it into something fundamentally different.
