## [07:55] Memory System Quick Wins — Shipped (commits 06cb208..7e8d0fc)

**Context:** Sub-agent `memory-analysis` produced full inventory + mempalace comparison report at `/tmp/memory-system-analysis.md`. Bob asked to ship the 4 quick wins from the report, then spawn follow-up for strategic items.

**Quick wins:**

1. **Hot cache key consistency** — Search-flow was caching LanceDB int IDs while get-flow looked up by UUID strings → 0% hit rate on repeated gets. Fix: lazy-loaded `_INT_TO_UUID` mapping in `LanceWarmStore`, populated on upsert + on-demand from SQLite via `MemoryDB.get_uuid_for_int_id()`. `_row_to_dict` now translates int → UUID before returning. Verified 100% hit rate on repeats.

2. **Weekly consolidation LanceDB resync** — `weekly-memory-consolidation.sh` now runs `python3 memory_tier_manager.py rebuild` after writing new memories. Kills the drift bug where SQLite gained rows but LanceDB never saw them.

3. **SHA-256 → 63-bit IDs** — Replaced `abs(hash(uuid)) % (2**31)` (31-bit, randomised per-process) with `int.from_bytes(hashlib.sha256(uuid.encode()).digest()[:8], 'big') & ((1<<63)-1)` (63-bit, deterministic). Helper: `_stable_int_id()` in both `memory_lance_store.py` and `memory_tier_manager.py`. Added collision-detection warning on sync. Migration: ran one-time clean-rebuild — truncated 465 stale rows, re-embedded 234 with new IDs.

4. **Cold archive time-decay** — Replaced flat `_score = 0.01` with `0.04 * 0.5^(age_days/180)` (180-day half-life, floor 0.005). Fresh archive 0.04, 6mo 0.02, 18mo 0.005. Old-but-relevant memories can now resurface.

**Bonus:** Added `LanceWarmStore.truncate()` and `--rebuild --clean` CLI flag for clean-rebuild support.

- `scripts/memory_lance_store.py` — `_stable_int_id`, `_INT_TO_UUID`, `_row_to_dict` translation, `truncate()`, sync_from_sqlite collision warning
- `scripts/memory_tier_manager.py` — `_stable_int_id`, `_cold_time_decay_score`, lazy mapping populate, store/delete/promote/demote SHA-256 IDs
