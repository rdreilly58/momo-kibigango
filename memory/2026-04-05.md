## Model Config Fix — claude-cli → anthropic (April 6, 5:49 AM) ✅

**Problem:** Both Telegram and Rocket.Chat returning `Budget 0 is invalid. This model only works in thinking mode.`

**Root Cause:** Two-step failure chain:
1. Default model was `claude-cli/claude-sonnet-4-6` — the `claude-cli` provider had **no auth configured** in 2026.4.2
2. Gateway fell back to `google/gemini-2.5-pro`, but with `thinking budget: 0` (off)
3. Gemini 2.5 Pro **requires thinking mode** — rejected with 400 error

**Fix:** Changed all `claude-cli/` model references to `anthropic/` in `openclaw.json`:
- `agents.defaults.model.primary`: `anthropic/claude-sonnet-4-6`
- `agents.defaults.model.fallbacks`: `["google/gemini-2.5-pro", "anthropic/claude-haiku-4-5"]`
- Removed `claude-cli/claude-opus-4-6` and `claude-cli/claude-sonnet-4-6` from models map
- Added `anthropic/claude-sonnet-4-6` and `anthropic/claude-opus-4-6` instead

**Key Lesson:** After updates, verify `openclaw models` shows auth for the default provider. `claude-cli` is not a real provider — use `anthropic/` for Anthropic models.

## Telegram Channel Fix (8:14 PM) ✅

**Problem:** Telegram stopped working after the 2026.4.2 gateway update. The bot was alive and messages were queued (7+ pending), but the gateway silently refused to start the telegram provider.

**Root Cause:** OpenClaw 2026.4.2 changed telegram from a **built-in channel** to a **plugin** that requires explicit registration. The old config only had `channels.telegram` settings, but the new version also needs:
- `plugins.allow` → must include `"telegram"`
- `plugins.entries.telegram` → must have `{"enabled": true}`

**Fix:**
```json
{
  "plugins": {
    "allow": ["openclaw-channel-rocketchat", "acpx", "telegram"],
    "entries": {
      "telegram": { "enabled": true }
    }
  }
}
```

**Key Lesson:** After OpenClaw updates, if a channel silently stops working, check if it was moved from built-in to plugin. The symptom is zero log lines for that channel on startup — not even an error.

**Detection Time:** ~20 minutes of debugging. The gateway was updated at 18:54 via `update.run`, and telegram worked under the old binary but failed on every restart after.
