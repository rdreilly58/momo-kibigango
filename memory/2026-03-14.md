### Root Cause Identified: Unnecessary PrismaAdapter

**File:** `src/app/api/auth/[...nextauth]/route.ts`

**The Problem:**
```typescript
export const authOptions: AuthOptions = {
  adapter: PrismaAdapter(prisma) as any,  // ← PROBLEM HERE
  providers: [GoogleProvider(...)],
  session: {
    strategy: "jwt",  // ← Using JWT, NOT database sessions
  },
};
```

**Why This Causes Hanging:**
1. Code uses `session: { strategy: "jwt" }` — JWT tokens store user data, don't need database
2. But ALSO uses `PrismaAdapter(prisma)` — designed for database sessions
3. When NextAuth initializes, `PrismaAdapter` tries to connect to Neon PostgreSQL
4. **Neon connection times out** (pool exhaustion or connectivity issue)
5. Entire auth route handler blocks waiting for database
6. Route times out after 30 seconds

**Why PrismaAdapter Is Unnecessary:**
- JWT sessions encode all user data in the token itself
- No database lookup needed per request
- User data (id, email, role) comes from JWT, not from database queries
- PrismaAdapter is dead code in this configuration

### The Fix (Super Simple)

**Remove 3 lines from `src/app/api/auth/[...nextauth]/route.ts`:**

```typescript
// DELETE THESE IMPORTS:
- import { PrismaAdapter } from "@auth/prisma-adapter";
- import { prisma } from "@/lib/prisma";

// DELETE THIS LINE IN authOptions:
- adapter: PrismaAdapter(prisma) as any,
```

**Everything else stays the same** — Google OAuth, JWT sessions, user roles, everything still works.

### Why This Works

**JWT Token Flow:**
1. User logs in with Google
2. NextAuth creates JWT token containing user data (id, email, role, image, etc.)
3. Token stored in secure HTTP-only cookie
4. On each request, JWT is verified (no database needed)
5. User data extracted from JWT payload

**Database Sessions (what PrismaAdapter is for):**
- User logs in, session stored in database
- Cookie contains only session ID
- On each request, session ID must be looked up in database
- More secure (server controls session), but slower

**Our Case:** JWT is perfect because:
- ✅ Fast (no database lookup per request)
- ✅ Stateless (doesn't need PrismaAdapter)
- ✅ Resilient (works even if database is down)
- ✅ Scalable (no database load for auth)
