## ⚠️ CONFIG PROTECTION RULES (ENFORCE GOING FORWARD)

**CRITICAL: Never modify `~/.openclaw/openclaw.json` without:**
1. Backing up first: `cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.backup`
2. Validating the result: check that ALL channel configs still exist (telegram, rocketchat)
3. Using `openclaw config set` for single-field changes (safer than full-file rewrites)
4. NEVER read whole file → modify in Python → write back (drops sections!)

**What went wrong:** A Python script read the JSON5 config, modified one field, and wrote it back as strict JSON — dropping the `channels.rocketchat` section entirely.

## Lessons Learned

1. **Multiple RC integrations = chaos.** Legacy standalone services + gateway plugin + RC webhooks all fighting.
2. **Docker healthcheck: test inside the container.** `curl` from host worked, but `curl` didn't exist inside.
3. **`localhost` doesn't always resolve in containers.** Use `127.0.0.1`.
4. **Plugin SDK version mismatches are silent killers.** Plugin connects fine, crashes on message handling.
5. **Config file rewrites are dangerous.** Use `openclaw config set` for single fields, never full-file Python rewrites.
6. **Approval flow is painful for rapid debugging.** Need exec approvals enabled on Telegram for future sessions.
