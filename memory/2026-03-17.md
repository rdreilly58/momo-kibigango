### Critical Lessons Learned

1. **Disk Strategy for ML:**
   - Root filesystem: 100GB minimum (system + caches)
   - Data volume: Separate, large (200GB+ for models)
   - Always mount /mnt/data before installation
   - Set PIP_CACHE_DIR and TMPDIR to data volume

2. **vLLM Limitations:**
   - GPU-first design (CUDA, Triton, cutlass)
   - CPU mode incomplete/untested in v0.17.1
   - Workaround: Use transformers directly for CPU
   - Plan: vLLM likely works better on GPU instances

3. **Model Selection for CPU:**
   - Large models (7B+): Need 28GB+ RAM
   - Quantized models: 4-8GB RAM (4-bit quantization)
   - TinyLlama-1.1B: 2-4GB RAM, 1 token/sec
   - Phi-2B: 4-8GB RAM, slightly better quality
   - Always test --dtype float32 on CPU (memory-efficient)

4. **Deployment Resilience:**
   - Volume expansion survives reboot ✅
   - /mnt/data requires remount (fix in /etc/fstab)
   - SSH key-based auth essential (password auth fails)
   - Background processes survive reboot (respawn from cron/systemd)
