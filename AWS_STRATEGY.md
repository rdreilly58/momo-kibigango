```bash
# Set CloudFront distribution for CDN
```

- Cheaper than Vercel for static assets
- Global CDN for portfolio images
- Automated backup system

---

#### **2️⃣ RDS (PostgreSQL) vs. Neon (Current)**
**Decision:** Keep Neon for now, use RDS if scaling

**Why Neon is better initially:**
- Serverless (auto-scaling)
- Cheaper at small scale
- Better DX

**When to switch to RDS:**
- Traffic > 10K users/day
- Need dedicated instance control
- Custom performance tuning

**For now:** Monitor via AWS CLI

### **Tier 2: Operational Excellence**

#### **4️⃣ AWS Lambda (Serverless Functions)**
**Cost:** Free tier (1M requests/month)

**Use Cases:**
- ✅ Image optimization pipeline
- ✅ Invoice generation API
- ✅ Scheduled tasks (backups, cleanup)
- ✅ Email notifications
- ✅ Webhook handlers (Stripe webhooks)

**Example Workflow:**
```
Stripe Payment → Lambda → Generate Invoice → S3 → Email to Client
```

**Benefit:**
- Scales automatically
- Pay-per-invocation (cheap)
- No server management

#### **5️⃣ SNS/SQS (Messaging & Queues)**
**Cost:** ~$0.50/month for small apps

**Use Cases:**
- ✅ Email notifications (payment receipts)
- ✅ Async job processing (invoice generation)
- ✅ Webhook queue (reliability)
- ✅ Alert system (deployment failures)

**Example:**
```
Invoice Request → SQS Queue → Lambda → S3 → SNS Email
```

**Benefit:**

- Automatic retry logic

## 🔗 AWS CLI Quick Reference

```bash
# List all S3 buckets
aws s3 ls

# Upload file to S3
aws s3 cp ./file.pdf s3://reillydesignstudio-assets/invoices/

# Get AWS account info
aws sts get-caller-identity

# List IAM users
aws iam list-users

# View cost data
aws ce get-cost-and-usage --time-period Start=2026-03-01,End=2026-03-31 \
  --granularity MONTHLY --metrics BlendedCost

# Create Lambda function (CLI)
aws lambda create-function --function-name invoice-generator \
  --runtime nodejs18.x --role arn:aws:iam::ACCOUNT:role/lambda-role

# Test deployment
aws amplify list-apps
```

## 🚨 Important: AWS Best Practices

**DO:**
- ✅ Use IAM roles (never use root credentials)
- ✅ Enable MFA
- ✅ Tag resources for organization
- ✅ Monitor costs weekly
- ✅ Set up automated backups
- ✅ Use VPC for security

**DON'T:**
- ❌ Share AWS credentials
- ❌ Hardcode API keys in code
- ❌ Disable CloudTrail
- ❌ Use wildcard (*) in IAM policies
- ❌ Leave security groups open
- ❌ Ignore cost alerts
